# FinTube - Browser Extensions

A small companion extension for [FinTube](../README.md). It reads your **YouTube cookies** from the
browser and formats them so you can paste them straight into FinTube's plugin settings in Jellyfin.

## Why?

yt-dlp (the downloader FinTube uses) sometimes fails on videos that are **age-restricted, private,
or members-only** - YouTube refuses to serve them without a logged-in session. Passing your browser
cookies to yt-dlp fixes that.

Exporting cookies by hand is fiddly. This extension does it in one click: open YouTube while logged
in, click the extension, hit **Copy cookies**, and paste the result into
**Jellyfin → Dashboard → Plugins → FinTube → Settings → YouTube Cookies**.

> The extension only ever copies cookies to your clipboard. It does **not** download anything, talk
> to your Jellyfin server, or send data anywhere.

## Builds

Prebuilt extensions are committed to this repo — just download the one for your browser, no cloning or
building required.

| Browser | Download | Source | Setup guide |
|---------|----------|--------|-------------|
| Firefox / LibreWolf / Zen / Floorp … | [`fintube-extension-firefox.xpi`](./fintube-extension-firefox.xpi) | [`firefox/`](./firefox) | [`firefox/README.md`](./firefox/README.md) |
| Chrome / Edge / Brave / Vivaldi … | [`fintube-extension-chrome.zip`](./fintube-extension-chrome.zip) | [`chrome/`](./chrome) | [`chrome/README.md`](./chrome/README.md) |

Both builds share the same popup, styling and cookie logic. They differ only in the way each browser
loads an MV3 extension (Firefox uses an add-on ID + background event page; Chromium uses a service
worker).

> [!NOTE]
> **Maintainers:** the `.xpi`/`.zip` archives are generated by [`build.sh`](../build.sh) in the repo
> root. After editing anything under `firefox/` or `chrome/`, re-run `./build.sh` and commit the
> updated archives so users always get the latest build.

## How to use

1. Install the build for your browser (see the guides above).
2. Log in to [youtube.com](https://www.youtube.com) in that browser.
3. Click the FinTube icon in the toolbar → **Copy cookies**.
4. In Jellyfin: **Dashboard → Plugins → FinTube → Settings**, paste into the **YouTube Cookies** box,
   and click **Save**.
5. Download as usual from the FinTube page. yt-dlp now uses your cookies.

> [!TIP]
> Cookies expire. If downloads that need a login start failing again, just re-export and paste a
> fresh set.

## Privacy

Cookies are read locally with the browser's `cookies` API and written only to your clipboard when you
click **Copy cookies**. Nothing is uploaded by the extension. Once pasted into FinTube, they live on
your own Jellyfin server (used to build the cookie file handed to yt-dlp).
